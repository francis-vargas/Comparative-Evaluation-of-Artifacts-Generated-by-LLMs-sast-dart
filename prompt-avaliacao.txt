PARTE 1 — EXECUÇÃO DE TESTES (OBRIGATÓRIO)
================================================================================

Para CADA uma das 4 versões, execute os seguintes passos:

### Passo 1: Clone o repositório
```bash
git clone https://github.com/francis-vargas/Comparative-Evaluation-of-Artifacts-Generated-by-LLMs-sast-dart.git
cd Comparative-Evaluation-of-Artifacts-Generated-by-LLMs-sast-dart

Passo 2: Para cada versão (gemini, gpt, grok, sonnet):

# Entre no diretório da versão
cd <nome-da-versao>

# Instale dependências (se houver requirements.txt ou pyproject.toml)
pip install -e .  # ou pip install -r requirements.txt

# Execute os testes
# Para unittest:
python3 -m unittest discover -s tests -v

# Para pytest:
pytest tests/ -v

# Documente o output COMPLETO

Passo 3: Teste a CLI manualmente

# Teste com arquivo de exemplo (se existir)
dart_sast tests/fixtures/vulnerable.dart

# Teste com diretório
dart_sast tests/fixtures/

# Teste formatos de saída
dart_sast tests/fixtures/ --format json
dart_sast tests/fixtures/ --format sarif

Passo 4: Verifique a estrutura do projeto

# Liste todos os arquivos
find . -type f | grep -v __pycache__ | sort

# Verifique presença de arquivos obrigatórios
ls -la README.md LICENSE Dockerfile pyproject.toml action.yml 2>/dev/null

SeloR (Reprodutibilidade)
Testes executam sem erros?
Resultados dos testes são consistentes?
Instruções permitem reprodução exata?
Sem dependências ocultas ou ambiguidades?
